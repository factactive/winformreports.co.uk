Release Notes: Changes in the Latest PrintForm, PrintControls and PrintAdapter Products

ID: W3298005

 

Background

This document details the main changes you should be aware of in the following releases:

  • PrintForm version 2.1.0.0
  • PrintAdapters version 2.1.0.0
  • PrintControls version 1.1.0.0

PrintForm 2.1.0.0

  1. Core printing engine completely rewritten. UseLegacyPrinting concept is now obsolete, printing of each control Type is assigned to a specific IPrintMethodAdapter implementation. New implementations can be registered with PrintForm to extend its capabilities, and existing implementations can be instructed to print additional Types (e.g. of 3rd party controls). See 'HowTo: Assign a PrintMethodAdapter to a Control in PrintForm.NET'.
  2. Updated to work with Microsoft Visual Studio 2019. Current build is against version 1.0 of the runtime so will work in 1.0, 1.1 or 2.0.
  3. Namespace and assembly name changes are detailed here. Namespaces and typenames have been unified across all three PrintForm, PrintAdapters and PrintControls products.

PrintAdapters 2.1.0.0

  1. Updated to work with Microsoft Visual Studio 2019. Current build is against version 1.0 of the runtime so will work in 1.0, 1.1 or 2.0.
  2. Namespace and assembly name changes are detailed here. Namespaces and typenames have been unified across all three PrintForm, PrintAdapters and PrintControls products.

PrintControls 1.1.0.0

  1. Updated to work with Microsoft Visual Studio 2019. Current build is against version 1.0 of the runtime so will work in 1.0, 1.1 or 2.0.
  2. Namespace and assembly name changes are detailed here. Namespaces and typenames have been unified across all three PrintForm, PrintAdapters and PrintControls products.
